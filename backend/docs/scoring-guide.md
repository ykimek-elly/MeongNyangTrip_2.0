# 추천 점수 계산 기준 정리

기준 코드:
- `backend/src/main/java/com/team/meongnyang/recommendation/service/PlaceScoringService.java`
- 실제 정렬 경로는 `scorePlaces(...) -> scorePlaceEntityDriven(...)`

## 1. 전체 점수 구조 (100점 기준)

| 구분 | 최대 점수 | 설명 |
| --- | ---: | --- |
| 펫 출입/정책 적합도 | 30점 | 반려동물 동반 가능 여부와 정책 명확성 |
| 반려동물 편의시설 적합도 | 15점 | 편의시설, 이용 편의, 안전/쾌적 |
| 장소 신뢰도/품질 | 20점 | 평점, 리뷰 수, AI 평점, 블로그 수 |
| 블로그 감성 신호 | 15점 | 긍정/부정 블로그 태그 |
| 태그/카테고리 적합도 | 20점 | 카테고리, 태그, 숙소 보정, 데이터 충실도 |
| 감점 | 최대 -15점 + 다양성 감점 | 제한 조건, 부정 후기, 최근 중복 추천 |
| 총점 | 100점 | `합산 - 감점`, 이후 0~100으로 보정 |

점수 공식:

```text
총점 = 정책 + 시설 + 품질 + 블로그 + 태그/카테고리 - 감점
```

## 2. 항목별 가중치

| 항목 | 비중 | 면접용 한 줄 설명 |
| --- | ---: | --- |
| 펫 출입/정책 적합도 | 30% | 실제로 데려갈 수 있는지 가장 중요해서 가장 크게 둠 |
| 장소 신뢰도/품질 | 20% | 평점과 리뷰 기반으로 기본 품질 검증 |
| 태그/카테고리 적합도 | 20% | 설명 가능한 태그와 메타데이터가 풍부한지 반영 |
| 반려동물 편의시설 적합도 | 15% | 동반 경험이 편한 장소인지 반영 |
| 블로그 감성 신호 | 15% | 실제 방문 체감에 가까운 후기 신호 반영 |

|  | 정보 없음/애매함 | 6 |

|  | 짧지만 정보 있음 | 5 |

### 3-2. 반려동물 편의시설 적합도 (15점)

| 안전/쾌적 | `안전`, `조용`, `청결`, `쾌적`, `관리` 매칭 수 x 0.7 | 최대 2 |

편의시설 키워드 예시:

### 3-3. 장소 신뢰도/품질 (20점)

| 세부 항목 | 기준 | 점수 |

| AI 보조 평점 | 4.5 이상 3점 / 4.0 이상 2.5점 / 3.5 이상 2점 / 3.0 이상 1점 / 그 미만 0.5점 | 최대 3 |
|  | AI 평점 없음 | 1 |
| 블로그 신뢰도 | 200개 이상 4점 / 80개 이상 3점 / 20개 이상 2점 / 5개 이상 1점 | 최대 4 |
| 추가 보정 | 리뷰 10개 미만이고 블로그 5개 미만이면 전체 품질 점수 x 0.8 | 감산 효과 |

### 3-4. 블로그 감성 신호 (15점)

| 세부 항목 | 기준 | 점수 |

|  | `냄새`, `청결불량`, `불친절` | 키워드당 2.0 감점 |

최종 계산:

```text
블로그 점수 = 5 + 긍정 점수 - 부정 감점
```

## 4. 감점 기준

### 4-1. 점수 계산 전 제외되는 경우

| 조건 | 처리 |
| --- | --- |
| `반려동물 출입 불가`, `애견동반 불가`, `동반 불가`, `출입 금지` 등 | 후보에서 제외 |
| `폐업`, `운영 종료`, `휴업` 등 | 후보에서 제외 |
| 숙소(`STAY`)인데 `accomCountPet`가 `0`, `불가`, `없음`, `미제공` | 후보에서 제외 |

### 4-2. 점수 계산 후 감점되는 경우

| 감점 항목 | 기준 | 감점 |
| --- | --- | ---: |
| 실내 제한 | `실내 불가`, `야외만 가능`, `실외만 가능` | -8 |
| 추가 제한 | `소형견만`, `대형견 불가`, `사전 문의`, `추가 요금` | -4 |
| 부정 후기 | `혼잡`, `붐빔`, `소음`, `냄새`, `청결불량`, `대기`, `불친절` | 키워드당 -1.5, 최대 -8 |
| 장소 데이터 감점 총합 |  | 최대 -15 |
| 다양성 감점 | 최근 추천 이력 기반 외부 penalty 적용 | 가변 |

### 4-3. 다양성 감점 처리

| 후보 수 | 처리 |
| --- | --- |
| 4개 이상 | 전달된 다양성 감점 그대로 적용 |
| 3개 | 감점 절반만 적용 |
| 2개 이하 | 다양성 감점 미적용 |

## 5. 정보 없음 처리 방식

| 상황 | 처리 방식 |
| --- | --- |
| 정책 정보 없음 | 0점이 아니라 중립 점수 적용. 예: 실내 동반 가능성 6점, 정책 명확성 2점 |
| 편의시설 정보 없음 | 기본 1점 부여 |
| AI 평점 없음 | 중립 보정 1점 |
| 블로그 긍정 태그 없음 | 추가 가점 없이 유지 |
| 블로그 부정 태그 없음 | 감점 없음 |
| 일부 메타데이터 비어 있음 | 데이터 충실도 점수만 낮아짐 |
| 후보 리스트 자체가 비어 있음 | 빈 리스트 반환 |

정리하면:
- 정보가 없다고 바로 탈락시키지 않음
- 다만 높은 점수를 받기 어렵게 설계함
- 명확한 금지 정보만 있으면 아예 후보에서 제외함

## 6. 실제 예시

### 예시 1. 점수가 높게 나오는 장소

조건:
- 실내 동반 가능 명시
- 정책 문구 구체적
- `물그릇`, `어메니티`, `산책로` 존재
- 평점 4.8, 리뷰 180개, AI 평점 4.4, 블로그 90개
- 긍정 태그 `청결`, `쾌적`, `조용`, `친절`
- 감점 요소 없음

| 항목 | 점수 예시 |
| --- | ---: |
| 펫 출입/정책 적합도 | 28.0 |
| 반려동물 편의시설 적합도 | 14.9 |
| 장소 신뢰도/품질 | 17.2 |
| 블로그 감성 신호 | 10.1 |
| 태그/카테고리 적합도 | 12.4 |
| 감점 | 0 |
| 총점 | 82.6 |

해석:
- 동반 가능성이 명확하고
- 시설/후기/메타데이터가 고르게 좋아서
- 설명 가능한 상위 추천 후보가 됨

### 예시 2. 점수가 낮게 나오는 장소

조건:
- 정책에 `실내 불가`, `사전 문의`
- 편의시설 정보 거의 없음
- 평점 4.1이지만 리뷰 3개, 블로그 1개
- 부정 태그 `혼잡`, `소음`

| 항목 | 점수 예시 |
| --- | ---: |
| 펫 출입/정책 적합도 | 14.0 |
| 반려동물 편의시설 적합도 | 1.0 |
| 장소 신뢰도/품질 | 6.0 |
| 블로그 감성 신호 | 2.0 |
| 태그/카테고리 적합도 | 7.1 |
| 감점 | -15.0 |
| 총점 | 15.1 |

해석:
- 완전 제외까지는 아니어도
- 실내 제한 + 사전 문의 + 부정 후기 때문에 점수가 크게 깎임

## 7. 면접에서 설명하는 방법

### 짧게 설명하는 버전

> 추천 점수는 100점 기준이고, 현재 실제 코드는 장소 메타데이터 중심으로 계산합니다.  
> 가장 큰 비중은 반려동물 출입 가능성과 정책 명확성 30점이고, 그 다음이 장소 품질 20점, 태그/카테고리 적합도 20점, 편의시설 15점, 블로그 감성 15점입니다.  
> 여기에 실내 불가, 추가 요금, 사전 문의, 부정 후기 같은 조건은 감점하고, 반려동물 출입 불가처럼 명확한 금지 정보가 있으면 아예 후보에서 제외합니다.  
> 정보가 없을 때는 0점 처리보다는 중립 점수를 줘서 탈락은 막되, 정보가 풍부한 장소가 더 높은 점수를 받도록 설계했습니다.

### 한 문장 요약

> "갈 수 있는 곳인지 먼저 보고, 그 다음에 편의시설과 후기 신뢰도를 점수화해서, 설명 가능한 방식으로 1등 장소를 고르는 구조입니다."
